# Contributing (RU)

Этот репозиторий принимает вклад через pull request.

## Короткий процесс

1. Создайте ветку (`feat/...`, `fix/...`, `chore/...`).
2. Вносите изменения в канонические источники (`docs/playbooks/*`, затем зеркальте в `project-files/docs/playbooks/*` при необходимости).
3. Запустите локальные проверки (`markdownlint`, `lychee`, shell checks при изменении shell-файлов).
4. Откройте PR и дождитесь зелёных checks.

## Важные правила

- Не дублируйте workflow-логику в обёртках.
- Не добавляйте runtime/build-зависимости в этот репозиторий.
- Для изменений процесса редактируйте playbook, а не wrapper.

Подробные правила см. в английской версии: [docs/CONTRIBUTING.md](CONTRIBUTING.md).
